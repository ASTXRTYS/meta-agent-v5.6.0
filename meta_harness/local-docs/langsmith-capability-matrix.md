# LangSmith Capability Matrix For Evaluation Evidence Workbench

Source-audited for `TICKET-001` on 2026-04-26. This is a `local-docs/` research artifact, not a product spec.

## Access path key

- `sdk_direct` — HE-owned script/backend code should call LangSmith/OpenEvals SDK directly.
- `cli_skill` — agent/operator may use CLI guidance; not product-critical.
- `policy_tool` — Meta Harness-owned policy boundary adds provenance, visibility, redaction, artifact registration, analytics validation, bounded export, or audit logging.
- `not_supported_v1` — unavailable, brittle, too risky, or unnecessary for v1.

## Summary decision

The Evidence Workbench should use a **hybrid access-path matrix**:

- **Primary raw capability layer:** `sdk_direct` for LangSmith SDK and OpenEvals calls.
- **Product safety layer:** `policy_tool` for bounded bundles, artifact registration, redaction, visibility, analytics validation, and audit records.
- **CLI layer:** `not_supported_v1` in this installed environment because no LangSmith CLI command surface exists.

| `2026-04-13` | `@Cascade` | Added `StagnationGuardMiddleware` to universal baseline (Q12 amendment). Custom middleware: two-tier progress-aware call limiter. Tier 1: at `check_interval` boundary, pluggable `ProgressSignal` providers check agent state for stagnation (todo completion, artifact production, tool-call repetition); if stagnant, inject `SystemMessage` nudge (zero additional model calls). Tier 2: after `grace_after_nudge` more calls with no improvement, hard stop via `jump_to=end`. Model-agnostic, middleware-agnostic, domain-agnostic — designed for LangChain contribution viability. Per-role thresholds (Developer 50, Researcher/HE 40, Architect 35, Planner/Evaluator 30, PM 20). Graceful absence: missing state keys → `progressing=True` with `confidence=0.0`. |
| `2026-04-13` | `@Cascade` | Added `CollapseMiddleware` + `ContextEditingMiddleware` to universal baseline (Q12 amendment). Documented layered context management pipeline: Layer 0 (free, every turn) Collapse → Layer 1 (free, every turn) ContextEditing → Layer 2 (1 LLM call, at threshold) SDK SummarizationMiddleware → Layer 3 (on-demand) SummarizationToolMiddleware. Collapse groups consecutive read/search tool calls into badge summaries (4.2x later summarization trigger). ContextEditing clears old tool results at token threshold (mirrors Anthropic `clear_tool_uses`). Both are free — no LLM calls. Decision: v1 keeps SDK's built-in `SummarizationMiddleware` (cannot disable without patching `create_deep_agent()`); v2 will contribute `disable_summarization` flag upstream and switch to `CompactionMiddleware` (compact-middleware, strict superset). `CompactionMiddleware` deferred to v2. |
| `2026-04-13` | `@Cascade` | Added `ModelCallLimitMiddleware` to universal baseline (Q12 amendment). SDK built-in, not included in `create_deep_agent()` by default. Counts model API calls (cost-accruing unit), not graph steps. Per-role `thread_limit`: Developer 500, Researcher/HE 300, Architect 250, Planner/Evaluator 200, PM 150. `run_limit=None` for all. `exit_behavior="end"`. Documented distinction from `recursion_limit` (SDK default 9,999 = graph safety valve, not cost control). v0.5 `recursion_limit=100` ≈ only 30–50 model calls — root cause of v0.5 limit issues. Composes with `StagnationGuardMiddleware`: hard ceiling (regardless of progress) + progress-aware limiter (spinning detection). Neither subsumes the other. |
